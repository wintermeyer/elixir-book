= Preface
Stefan Wintermeyer <sw@wintermeyer-consulting.de>

This book teaches the **basics of Elixir**: the language
fundamentals you need before you pick up Phoenix, Ash, Nerves, or
any other BEAM framework. Pattern matching, the pipe operator,
processes, modules and functions, higher-order functions, data
types, control structures, recursion, Mix — the bedrock.

The material started life as the Elixir chapter of the
link:https://wintermeyer-consulting.de/phoenix/book/[_Elixir, Phoenix and
Ash Beginner's Guide_]. It grew large enough to stand on its own
and became a separate book. The Phoenix guide still points here
for the language part; this book stays focused on the
fundamentals and doesn't touch web frameworks.

The sidebar lets you jump to any topic at any time — nothing is
locked, and later chapters never assume you read the preceding
ones to the letter. That said, the book is written to be read
front to back: each topic sets up the next, examples build on
earlier ones, and the shortest path from zero to confident
Elixir is still "start at the top and keep going".

If this is your first functional programming language you might
want to get a cup of coffee first. It might take a while to get
used to the functional programming paradigm. **It took me a long
time too!**

== Further reading

Once the basics here click and you want to go deeper, the
official language documentation at
link:https://hexdocs.pm/elixir/introduction.html[hexdocs.pm/elixir]
is the natural next step. It covers the same ground and much
more, written by the people who build Elixir.
