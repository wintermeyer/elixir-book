= Preface
Stefan Wintermeyer <sw@wintermeyer-consulting.de>

This is a beginner's book for the **basics of Elixir**: the
language fundamentals you need before you pick up Phoenix, Ash,
Nerves, or any other BEAM framework. Pattern matching, the pipe
operator, processes, modules and functions, higher-order
functions, data types, control structures, recursion, Mix —
the bedrock.

I'll not sugar-coat this: Elixir is a functional language,
which means it thinks differently than the C / Java / Python /
Ruby-shaped one you probably already know. The paradigm takes
a while to settle. Grab a coffee first if this is your first
functional language — it took me a long time too. The upside:
once it clicks, it clicks hard.

The book is written the way I'd teach these basics in a
workshop — bite-sized examples, one concept at a time — so
you can work through it on your own at the same pace.

The material started life as the Elixir chapter of the
link:https://wintermeyer-consulting.de/phoenix/book/[_Phoenix_]
book. It grew large enough to stand on its own and became a
separate book. When you've got the language under your hands,
the Phoenix guide is where the web framework picks up.

The sidebar lets you jump to any topic at any time — nothing
is locked, and later chapters never assume you read the
preceding ones to the letter. That said, the book is written
to be read front to back: each topic sets up the next, examples
build on earlier ones, and the shortest path from zero to
confident Elixir is still "start at the top and keep going".

== Further reading

Once the basics here click and you want to go deeper, the
official language documentation at
link:https://hexdocs.pm/elixir/introduction.html[hexdocs.pm/elixir]
is the natural next step. It covers the same ground and much
more, written by the people who build Elixir.
